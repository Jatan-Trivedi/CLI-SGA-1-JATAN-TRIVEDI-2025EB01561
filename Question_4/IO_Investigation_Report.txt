2. Output redirection and error redirection results
Commands:
ls -l /etc/passwd /fake_directory > success.txt 2> error.txt
cat success.txt
cat error.txt
Output:
jatan_rivedi@LAPTOP-JR42J90G:~$ cat success.txt
-rw-r--r-- 1 root root 1438 Aug 13  2025 /etc/passwd
jatan_rivedi@LAPTOP-JR42J90G:~$ cat error.txt

Observation: By using `>`, I successfully redirected the Standard Output (File Descriptor 1) of the valid `/etc/passwd` file lookup into `success.txt`. By using `2>`, I redirected the Standard Error (File Descriptor 2) generated by the missing `/fake_directory` into `error.txt`. This completely separates successful data from error logs.

4. Explanation of how Linux manages file I/O
Linux manages file I/O based on the philosophy that "everything is a file," including hardware devices, directories, and standard text files. When a process interacts with a file, Linux assigns it an integer known as a File Descriptor (FD). Every process starts with three default FDs: 0 for Standard Input (stdin), 1 for Standard Output (stdout), and 2 for Standard Error (stderr). This abstraction allows developers to easily redirect data streams and errors to different files or devices using simple command-line operators like `>` and `2>`.
